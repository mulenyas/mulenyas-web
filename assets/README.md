# Assets / Imágenes

Esta carpeta debe contener todas las imágenes de la web.

## Imágenes necesarias:

### Hero y escenas
1. **hero.jpg** (1920x1200px mínimo)
   - Imagen principal editorial
   - Vertical u horizontal
   - Alta calidad
   - Ejemplo: modelo con mulanyas en escena mediterránea

2. **craft-detail.jpg** (1200x1500px)
   - Detalle de artesanía
   - Close-up de rafia, manos tejiendo, etc.

### Productos (The Object)
3. **mule-1.jpg** (1000x1333px - ratio 3:4)
   - Rafia Natural
   - Fondo limpio

4. **mule-2.jpg** (1000x1333px)
   - Yute con Flecos
   - Fondo limpio

5. **mule-3.jpg** (1000x1333px)
   - Rafia Oscura
   - Fondo limpio

### Story
6. **story-image.jpg** (1600x1000px)
   - Imagen editorial para el manifiesto
   - Puede ser lifestyle, detalle, escena

### Craft
7. **hands-weaving.jpg** (1200x1500px)
   - Manos tejiendo
   - Proceso artesanal

8. **materials.jpg** (1200x1500px)
   - Materiales: rafia, yute, esparto
   - Composición de texturas

9. **workshop.jpg** (1200x1500px)
   - Taller
   - Espacio de trabajo

## Consejos para las fotos:

### Estilo fotográfico
- Luz natural, cálida
- Colores suaves (marfil, arena, terracota)
- Fondos limpios o mediterráneos
- Nada de fondos blancos perfectos (muy e-commerce)

### Referencias visuales
- **Loewe:** craft shots, manos, detalles
- **Miu Miu:** editorial, color, juventud
- **Paloma Wool:** escenas, naturalidad
- **Jacquemus:** Mediterráneo, luz, calor

### Optimización
- Formato: JPG
- Peso máximo: 500KB por imagen
- Usa [TinyPNG](https://tinypng.com/) para comprimir
- Mantén alta calidad visual

### Nombrado
- Usa exactamente estos nombres
- Todo en minúsculas
- Sin espacios ni caracteres especiales
- Formato: `.jpg` (no .jpeg ni .JPG)

## Cómo subir las imágenes:

1. Optimiza las imágenes con TinyPNG
2. Renombra con los nombres exactos de arriba
3. Súbelas a esta carpeta `assets/`
4. Commit y push a GitHub

```bash
git add assets/
git commit -m "Añadir imágenes"
git push
```

---

**IMPORTANTE:** Sin estas imágenes, la web mostrará imágenes rotas. Es el último paso antes de que la web esté completamente funcional.
